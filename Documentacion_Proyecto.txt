ESTRUCTURA DEL PROYECTO Y ARCHIVOS

Directorio principal: c:\Users\gl391\Downloads\Opcion1

1. Archivos en la raíz del proyecto:
- .gitignore: Archivo para decirle a Git (control de versiones) qué archivos o carpetas ignorar para no subirlos al repositorio.
- README.md: Archivo de texto con formato Markdown para documentar información general del proyecto.
- eslint.config.js: Archivo de configuración de ESLint para el análisis de código (linting) y mantener buenas prácticas de programación.
- index.html: Archivo HTML principal de la aplicación donde se inyecta el código de React.
- package.json y package-lock.json: Archivos de configuración de npm que listan las dependencias (librerías) del proyecto y scripts de ejecución.
- postcss.config.js: Configuración para PostCSS, utilizado generalmente en conjunto con Tailwind CSS para procesar los estilos.
- tailwind.config.js: Archivo de configuración de Tailwind CSS (colores, fuentes, plugins, etc.).
- vite.config.js: Archivo de configuración de Vite, el empaquetador (bundler) de alta velocidad utilizado para el desarrollo y construcción del proyecto.

2. Carpetas en la raíz del proyecto:
- node_modules/: Carpeta generada automáticamente que contiene todas las dependencias y librerías instaladas del proyecto. No se debe modificar manualmente.
- public/: Contiene archivos públicos que no son procesados por Vite. Se sirven tal cual en el navegador.
  - favicon.svg: Icono de la pestaña del navegador.
  - icons.svg: Archivo que contiene iconos en formato SVG.
  - images/: Carpeta designada para guardar las imágenes públicas.
    - devlog/: Contiene las imágenes organizadas por semanas para los devlogs.
    - hero/: Contiene las imágenes para la sección principal (hero carousel).
- src/: Carpeta principal que contiene todo el código fuente de React.
  - App.jsx y App.css: Componente principal de React que envuelve la aplicación y sus estilos básicos.
  - main.jsx: Punto de entrada principal de la aplicación React donde se renderiza la app en el index.html.
  - index.css: Archivo de estilos globales donde se importan las directivas base de Tailwind CSS.
  - assets/: Carpeta para imágenes u otros recursos estáticos que sí son procesados por Vite (a diferencia de public).
  - components/: Carpeta que contiene todos los componentes reutilizables de React.
    - devlog/: Componentes específicos para la sección de los logs (DevLogCard.jsx, DevLogCarousel.jsx, DevLogModal.jsx).
    - header/: Componentes para la barra de navegación superior (Header.jsx).
    - hero/: Componentes para la sección inicial destacada (HeroCarousel.jsx).
    - weekSelector/: Componente para el menú selector de semanas (WeekSelector.jsx).
  - data/: Carpeta para almacenar datos estáticos que simulan una base de datos.
    - devlogData.js: Archivo crítico que contiene toda la información de las semanas, títulos, descripciones, rutas de imágenes y textos de los devlogs.


=============================================================================
INSTRUCCIONES PARA AÑADIR UNA NUEVA SEMANA Y CARTAS AL DEVLOG
=============================================================================

Toda la información que se muestra en la sección de Devlogs se controla desde un único archivo.
Sigue estos pasos para añadir contenido:

1. Abre el archivo: `src/data/devlogData.js`
2. Dentro verás una variable llamada `devlogData` que es una lista (array) de objetos, donde cada objeto es una semana.
3. Para añadir una nueva semana (por ejemplo, la semana 4), ve al final de la lista, añade una coma `,` después de la última llave `}` y pega una nueva estructura como esta:

  {
    week: 4,
    title: "El Nuevo Comienzo",
    cards: [
      {
        id: "w4-c1",
        title: "Diseño de la nueva zona",
        description: "Breve descripción de lo que se hizo en esta primera carta.",
        image: "/images/devlog/week4/mi_imagen_1.jpg",
        fullText: "Aquí va el texto completo y detallado que aparecerá cuando el usuario haga clic en la carta y se abra la ventana modal."
      },
      {
        id: "w4-c2",
        title: "Animaciones de ataque",
        description: "Mejorando el sistema de combate.",
        image: "/images/devlog/week4/mi_imagen_2.jpg",
        fullText: "Texto extendido explicando el proceso de creación de animaciones de ataque..."
      }
      // Puedes seguir añadiendo más cartas (objetos) aquí adentro, separándolos por comas
    ]
  }

Nota: Asegúrate de que los `id` de las cartas sean únicos (por ejemplo, "w4-c1", "w4-c2" para la semana 4).


=============================================================================
DÓNDE Y CÓMO AÑADIR LAS IMÁGENES
=============================================================================

Para que las imágenes se carguen correctamente y el proyecto esté ordenado, debes seguir estas reglas:

1. Dónde guardarlas en tus carpetas:
   - Navega a la carpeta `public/images/devlog/` en tu proyecto.
   - Crea una nueva subcarpeta para la semana que estás añadiendo (por ejemplo: `week4`).
   - Pega ahí tus archivos de imagen (ej. `mi_imagen_1.jpg`, `captura.png`, etc.).

2. Cómo enlazarlas en el código:
   - Cuando edites el archivo `src/data/devlogData.js` y estés configurando una carta, debes poner la ruta en la propiedad `image`.
   - La ruta debe comenzar siempre con `/images/` (ya que la carpeta public es la raíz para el navegador).
   - Ejemplo: Si guardaste tu imagen en `public/images/devlog/week4/foto.jpg`, en el código debes escribir exactamente:
     image: "/images/devlog/week4/foto.jpg"
